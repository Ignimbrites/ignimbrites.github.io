how it works

The platform is the framework

ignimbrites is a bet that, for a lot of sites, the modern web platform is already enough. No build tools, no dependencies, no abstraction layer to learn or replace later — just files a browser can run directly.

The whole thing is static files served as-is. There's no server-side code, no compile step, and no node_modules. You can read every line that ships, because what ships is exactly what's written. Here's how the moving parts fit together.

Routing without a server

A static host hands back a file that exists at a path; ask for a path with no file and you get a 404. So routes live after the # instead — the one part of a URL the server never sees. Every request resolves to index.html, and a small router reads the hash, matches it against a table, and swaps the contents of <main>. Plain <a href='#/about'> links just work, because the browser fires a hashchange event the router listens for.

For the rare case of someone typing a real path directly, a tiny 404.html turns that path into a hash and bounces to the root.

Composing from plain HTML

The header and footer live once, as fragments in partials/, and a ~20-line custom element pulls them in: 

<html-include src='partials/header.html'></html-include>

It fetches the file and injects it as ordinary page content, so the global stylesheet styles it like everything else. Page bodies work the same way — the router fetches pages/*.html and composes them into <main>.

A blog from one JSON file

Every post lives in data/posts.json, imported straight into a module with an import attribute — no fetch(), no JSON.parse(): 

import posts from '../data/posts.json' with { type: 'json' };

The list view and the article view import the same file; the browser fetches it once and shares it. Adding a post means editing one file and pushing.

Transitions, built in

Navigation animates with the View Transitions API: the router wraps each DOM swap in document.startViewTransition(), and the browser cross-fades between views for free. Matching view-transition-name values let a post's title morph out of the list and into the article. When prefers-reduced-motion is set, the router skips the effect entirely.

Styling that composes

A single <link> loads styles/index.css, which uses @layer and @import to pull in the rest in a fixed order. Cascade layers mean each stylesheet can be edited on its own without specificity battles, and the whole palette is a handful of oklch custom properties — including a light theme and a dark one.

What it runs on

Everything here is Baseline in current ignimbrites browsers. JSON module imports landed in Chrome 123, Safari 17.2, and Firefox 128. Same-document view transitions are supported in Chrome 111, Safari 18, and Firefox 144. Because that last feature isn't available cross-document in Firefox yet, the site deliberately stays in one document and routes with the hash.

Fewer moving parts isn't a limitation to apologise for. It's the feature.

Make it yours

Add a page by dropping an HTML fragment in pages/, adding a line to js/routes.js, and linking to it. Add a post by editing data/posts.json. Retune the look from styles/tokens.css. Then push it to any static host. The build log walks through each technique in more detail.
