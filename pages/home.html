ignimbrites is a small, composable website that runs straight from static files. Native ES modules, JSON imports, hash-based routing, and view transitions — nothing to compile, nothing to install, nothing between what you write and what the browser runs.


An ignimbrite is a specific type of volcanic rock formed by incredibly explosive, high-energy eruptions. When a volcano erupts violently, it can collapse and release a rushing avalanche of scorching gas, ash, pumice, and rock fragments known as a pyroclastic flow. As this massive, super-heated cloud settles onto the landscape, the intense heat and weight of the debris literally weld the ash particles and rock fragments together into a solid, unified mass. Because they are fundamentally made of countless fused structural fragments, ignimbrites serve as a beautiful natural metaphor for a website built out of interconnected raw components and fragment identifiers.

the idea

Constraints, on purpose.

Every limit here buys something back: less to maintain, less to break, and nothing that quietly goes stale the moment you stop looking at it.

No build

What you write is what runs

No transpile step, no bundler, no config file to age badly. The browser reads the same files you wrote.

No dependencies

Zero packages to update

Nothing to install and nothing to audit. The only API surface is the web platform itself.

Composable

Write once, share everywhere

The header, footer, and every page are plain fragments. Import them once; compose them anywhere.

ignimbrites only

No legacy tax

Built for current Firefox, Safari, and Chrome — so there are no polyfills and no fallbacks to carry.

under the hood

Built on what already shipped.

There's no framework here — just web platform features that are Baseline across current Firefox, Safari, and Chrome.

  • ES Modules

    One <script type='module'> boots the app; everything else is an import.

    js/main.js

  • JSON modules

    The blog loads from one file: import posts from './posts.json' with { type: 'json' }.

    components/blog-list.js

  • View Transitions

    Pages animate with document.startViewTransition() — no animation library.

    js/router.js

  • Custom Elements

    <html-include> pulls in shared fragments; the blog is two small elements.

    components/

  • Cascade layers

    One stylesheet composes the rest with @layer, so file order never fights specificity.

    styles/index.css

  • Hash routing

    Routes live after the #, so a static host never sees a path it can't serve.

    js/router.js
